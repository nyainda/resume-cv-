# CV AI Generation Rules
## Anti-Detection & Humanization Guidelines

---

## 1. VOICE & TONE RULES

### ❌ NEVER DO
- Use corporate filler phrases:
  - "passion for participating in brainstorming sessions"
  - "drive meaningful change through innovative technology"
  - "leveraging expertise to deliver value"
  - "results-driven professional"
  - "dynamic team player"
- Write every sentence in perfect formal grammar
- Use the same sentence structure repeatedly
- Sound like a legal document or press release

### ✅ ALWAYS DO
- Write like the candidate actually talks (professional but human)
- Vary sentence length — mix short punchy sentences with longer ones
- Include ONE slightly informal phrase per section (not every line)
- Use contractions occasionally: "didn't", "wasn't", "couldn't"
- Let personality show in the summary paragraph

---

## 2. METRICS & ACHIEVEMENTS RULES

### ❌ NEVER DO
- Put exactly 2-3 metrics in EVERY bullet point
- Use round numbers exclusively: 25%, 30%, 50%, 40%
- Make every metric sound like a marketing claim
- Use the same metric types repeatedly (always "% increase", "% reduction")

### ✅ ALWAYS DO
- Vary metric presence: some bullets have metrics, some don't
- Use oddly specific numbers occasionally: "7.5 hours/week", "saved ~3 days per sprint"
- Mix metric types: time saved, cost, user count, error count, satisfaction scores
- Leave 1-2 bullets as purely qualitative (no numbers)
- If a metric is estimated, say so: "reduced onboarding time by roughly 30%"

---

## 3. KEYWORD DENSITY RULES

### ❌ NEVER DO
- Repeat the same keyword more than 3 times across the whole CV
- Mirror the JD's exact phrases verbatim back at them
- Stuff every section with the same buzzwords
- Aim for 95-100% keyword match with the JD

### ✅ ALWAYS DO
- Target 65-75% keyword match — enough to pass ATS, not enough to look gamed
- Prioritize HARD SKILL keywords (tech stack, tools, methodologies)
- Deprioritize SOFT SKILL keywords (they're generic anyway)
- Distribute keywords naturally across different sections
- Rephrase JD keywords: if JD says "regulatory compliance", CV says "compliance requirements" or "audit-ready systems"

---

## 4. BULLET POINT STRUCTURE RULES

### ❌ NEVER DO
- Start every bullet with "Spearheaded", "Leveraged", "Utilized", "Facilitated"
- Make every bullet follow the exact same formula
- Write bullets longer than 2 lines
- Have the same number of bullets in every job section

### ✅ ALWAYS DO
- Vary opening verbs: Built, Wrote, Fixed, Reduced, Shipped, Designed, Cut, Helped, Led, Debugged
- Mix bullet formats:
  - Action + Result: "Rebuilt the auth system, cutting login failures by 40%"
  - Action + Context: "Wrote the API integration layer during a tight 2-week sprint"
  - Pure statement: "Sole backend developer on the v2 rewrite"
- Allow 3-5 bullets per role (not always the same count)

---

## 5. SUMMARY/PROFILE RULES

### ❌ NEVER DO
- Write a generic summary that could apply to any candidate
- List every technology in the summary
- Use more than 4 sentences
- Repeat what's already in the experience section

### ✅ ALWAYS DO
- Open with something specific to THIS candidate's story
- Mention one niche or unexpected skill/angle
- Keep it to 2-3 sentences max
- End with what they're actually looking for (forward-looking)

**Example (bad):**
> "Highly motivated software engineer with 2 years of experience leveraging expertise in regulatory compliance and GovTech to drive meaningful change through innovative technology..."

**Example (good):**
> "Backend engineer with 2 years building SaaS products, mostly in Laravel and React. I've shipped features used by government agencies and spent a lot of time making sure the data layer is actually secure, not just compliant on paper. Looking to join a team where the technical bar is high."

---

## 6. FORMATTING RULES

### ❌ NEVER DO
- Use identical date ranges formatting across all roles
- List 15+ skills in a flat bullet list
- Include every technology ever touched

### ✅ ALWAYS DO
- Group skills meaningfully: Frontend / Backend / DevOps / Tools
- Only list skills you can actually be interviewed on
- Keep skill list to 10-12 maximum
- Include one "currently learning" item — signals growth, sounds human

---

## 7. THE HUMANIZATION CHECKLIST
Run this before finalizing any generated CV:

- [ ] Does the summary sound like a real person wrote it?
- [ ] Are there any repeated phrases (3+ times)? Remove them
- [ ] Do ALL bullets have metrics? Remove metrics from 2-3 of them
- [ ] Are ALL metrics round numbers? Make 1-2 oddly specific
- [ ] Does any sentence appear in the JD word-for-word? Rephrase it
- [ ] Is keyword match above 80%? Deliberately remove some keywords
- [ ] Are there 3+ "Spearheaded/Leveraged/Utilized" openers? Replace them
- [ ] Would a recruiter reading this out loud sound like a robot? Rewrite those parts

---

## 8. KEYWORD TARGETING STRATEGY

### Priority Tiers:

**Tier 1 — MUST HIT (hard requirements):**
- Specific technologies mentioned (e.g. MySQL, React, Laravel)
- Years of experience requirements
- Degree/qualification requirements

**Tier 2 — SHOULD HIT (core responsibilities):**
- Main job function keywords
- Industry-specific terms
- Methodology keywords (Agile, MVC, REST, etc.)

**Tier 3 — SKIP (fluffy/generic):**
- "Team player", "self-starter", "detail-oriented"
- Vague soft skills
- Company mission/values language from the JD

### Keyword Placement Priority:
1. Job title line
2. Summary (1-2 Tier 1 keywords max)
3. Most recent role bullets
4. Skills section
5. Older roles (lighter touch)

---

## 9. AI DETECTION BYPASS SUMMARY

| Signal | AI CV | Human CV | Target |
|--------|-------|----------|--------|
| Keyword match | 90-100% | 40-60% | **65-75%** |
| Metric per bullet | 100% | ~40% | **50-60%** |
| Round numbers | 100% | ~30% | **50-60%** |
| Repeated phrases | High | Low | **Very low** |
| Sentence variety | Low | High | **High** |
| Grammar perfection | 100% | ~85% | **~90%** |

---

*These rules apply to the CV text generation stage only. Embedded/metadata layers are handled separately.*
