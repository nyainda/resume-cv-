- [x] **H** — Multi-token / role-based admin auth. New table `cv_admin_tokens(id, token_hash, label, role, created_at, last_used_at, revoked_at)` (migration `007_admin_tokens.sql`, applied to live D1). Tokens are stored as SHA-256 hashes — plaintext (`cvk_` + 64 hex chars) is returned exactly **once** at creation. Roles are hierarchical: `viewer` (read-only stats, browse, voice-test, AI audit, leak queue), `editor` (everything viewers can do + bulk add/update/delete + KV sync + leak decisions), `admin` (everything + token CRUD). New helper `verifyAdminAuth(req,env,role)` replaces the old inline `env.ADMIN_TOKEN === token` check across all 10 admin handlers; bootstrap `env.ADMIN_TOKEN` secret still works as a permanent admin so we never lock ourselves out. Worker routes `GET /api/cv/admin/tokens`, `POST /api/cv/admin/tokens` (`{label, role}`), `POST /api/cv/admin/tokens/revoke` (`{ids}`) — last guard prevents revoking the token you're currently authenticated with. New admin tab "Tokens" lists everyone, lets admins mint per-person tokens with role badges, copy-to-clipboard the one-time plaintext, and revoke. `last_used_at` is best-effort updated on every authenticated request via a non-blocking `UPDATE`. Worker redeployed (version `22f1bd85`).
